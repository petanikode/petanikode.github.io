Tutorial Composer untuk Pemula

Ardianta Pargo dalam pemrograman
26 November 2016 · 13:02

Composer merupakan sebuah alat (tool) untuk manajemen pustaka (library) dalam proyek web PHP. Composer dapat kita suruh melakukan instalasi dan update pustaka yang dibutuhkan. Kalau tidak pakai Composer, kita biasanya mengunduh sendiri pustakanya, kemudian di-copy ke direktori proyek. Bagaimana kalau banyak pustaka yang dibutuhkan, pasti kita akan melelahkan mengunduh dan menyalin semuanya. Oleh karena itu, kita harus menggunakan Composer.

1. Instalasi Composer

Panduan instalasi Composer sebenarnya sudah lengkap di dokumentasi Composer. Namun, saya ingin menunjukkan cara saya menginstalnya di Linux.

Instalasi melalui apt: sudo apt install composer

Pada distro Linux yang saya gunakan, paket Composer sudah tersedia di repositori. Akan tetapi masih versi beta (1.0.0-beta). Pada saat tulisan ini dubuat, versi terakhir Composer adalah 1.3-dev. Bila ingin menginstal versi terakhir, bisa menggunakan perintah berikut ini.
wget https://getcomposer.org/composer.phar
sudo cp composer.phar /usr/local/bin/composer
sudo chmod +x /usr/local/bin/composer
Kemudian coba periksa versi yang terinstal:
petanikode@imajinasi ~ $ composer –version
Composer version 1.2.2 2016-11-03 17:43:15

2. Menggunakan Composer

Sekarang saya punya sebuah proyek yang berlokasi di /var/www/html/belajar/belajar-composer. Saya perlu pindah dulu ke direktori tersebut.
cd  /var/www/html/belajar/belajar-composer
Kemudian ketik:
composer init
Setelah itu, silahkan isi informasi proyek seperti berikut ini.

Perintah di atas akan menghasilkan file composer.json:
{
    “name”: “petanikode/belajar-composer”,
    “description”: “Hanya untuk latihan composer saja”,
    “type”: “project”,
    “license”: “GPL”,
    “authors”: [
        {
            “name”: “Ardianta Pargo”,
            “email”: “ardianta_pargo@yahoo.co.id”
        }
    ],
    “require”: {}
}
Informasi yang tertera pada file composer.json merupakan informasi yang baru saja kita masukkan. Perhatikan pada atribut “require”, isinya masih kosong, karena tadi kita menjawab no pada define depedency.

Selanjutnya cobalah instal pustaka Carbon, pustaka ini menyediakan fungsi-fungsi untuk pengolahan waktu (Date Time). Ketik pada terminal:
composer require nesbot/carbon
Maka sekarang file composer.json akan menjadi seperti ini:
{
    “name”: “petanikode/belajar-composer”,
    “description”: “Hanya untuk latihan composer saja”,
    “type”: “project”,
    “license”: “GPL”,
    “authors”: [
        {
            “name”: “Ardianta Pargo”,
            “email”: “ardianta_pargo@yahoo.co.id”
        }
    ],
    “require”: {
        “nesbot/carbon”: “^1.21”
    }
}
Selain itu juga, perintah tersebut melakukan pengunduhan pustaka ke dalam proyek. Cobalah buka direktori proyeknya, pasti ada direktori baru bernama vendor. Semua pustaka yang diunduh Composer akan tersimpan di sana.
Selanjutnya, kita akan mencoba pustaka yang baru saja kita tambahkan. Buatlah file PHP baru di direktori proyek beranam tes_carbon.php dengan isi sebagai berikut.
<?php

require ‘vendor/autoload.php’;

use Carbon\Carbon;

$sekarang = Carbon::now();

echo “Sekarang: $sekarang <br>”;
echo “Umur saya: “ . Carbon::createFromDate(2000, 1, 1)->age . “<br>”;
echo “Besok: “ . $sekarang->addDay() .”<br>”;

?>
Hasil outputnya:
Sekarang: 2016-11-25 21:03:18 
Umur saya: 16
Besok: 2016-11-26 21:03:18

3. Mencari Pustaka

Misalkan kita membutuhkan lebih banyak pustaka untuk diinstal. Kita hanya perlu menambahkan nama-nama pustakanya ke dalam atribut “require”.
{
    “name”: “petanikode/belajar-composer”,
    “description”: “Hanya untuk latihan composer saja”,
    “type”: “project”,
    “license”: “GPL”,
    “authors”: [
        {
            “name”: “Ardianta Pargo”,
            “email”: “ardianta_pargo@yahoo.co.id”
        }
    ],
    “require”: {
        “nesbot/carbon”: “^1.21”,
        “setasign/fpdf”: “1.8.1” 
    }
}
Formatnya, “vendor/pustaka”: “versi”. Lalu, bagaimana cara kita mengetahui nama-nama vendor dari pustaka tersebut? Kita dapat menggunakan perintah:
composer search fpdf
Perintah tersebut untuk mencari nama vendor penyedia pustaka fpdf. Atau kita juga bisa mencarinya di website packagist.org.

Setelah semua pustaka yang kita butuhkan ditulis dalam file composer.json, selanjutnya kita bisa menginstal semuanya dengan perintah:
composer update

Penutup

Cukup sekian tutorial cara menggunakan Composer ini. Selengkapnya bisa dipelajari pada dokumentasi Composer atau kalau ada yang masih belum jelas, bisa ditanyakan melalui komentar.
