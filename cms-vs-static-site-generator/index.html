CMS vs Static Site Generator, Pilih yang Mana?

Pada umumnya, kebanyakan web—seperti blog—dibuat dengan CMS (Content Management System) seperti seperti Wordpress, Joomla, Blogger, Drupal, dsb.

Tapi, belakangan ini cukup banyak dari kalangan programmer yang ngeblog dengan Static Site Generator (SSG).

Entah apa pun alasannya…

Saya juga jadi ikut tertarik mencobanya…

Apa itu Static Site Generator?

Untuk menjawab pertanyaan ini, saya ingin membandingkan cara kerja web dinamis dan statis dulu.

Baiklah!

Cara kerja web dinamis bisa diumpamakan seperti cafe dan web statis seperti mesin penjual otomatis.

Misalnya begini, kita ingin pesan minuman di cafe. Lalu kita memilih menu dan memberitahukannya ke pelayan.

Setelah itu, minuman dibuat di dapur dengan bahan-bahan yang sudah ada.

Kita menunggu sampai minumannya jadi. Setelah itu, barulah kita bisa menikmati minumannya.

Pertama pengunjung memnita halaman (pesan minuman) melalui HTTP REQUEST, kemudian server membuatkan halamanya.

PHP memproses permintaan berdasarkan data yang ada di database (bahan baku minuman). Setelah itu PHP akan membuat output berupa HTML (minuman jadi) dan langsung dikirim ke pengunjung.

Pengunjung pun dapat menikmati halaman yang dimintanya.

Lalu bagaimana dengan web statis?

Seperti yang saya bilang tadi, cara kerja web statis mirip seperti mesin penjual otomatis.

Pengunjung meminta halaman, lalu server akan langsung memberikannya, tanpa harus diracik dengan database.

Artinya, halaman yang diminta sudah jadi dan tidak perlu dibikin dari awal.

Begitulah cara kerja web statis.

Balik lagi ke pertanyaan: Apa itu Static Web Generator?

Static Web Generator adalah mersin (engine) yang memproduksi halam web statis. Biasanya bahan bakunya dari Markdown dan plain text.

Perbedaan CMS dengan Static Web Generator

Salah satu perbedaan yang paling mencolok antara CMS dan SSG adalah penggunaan bahan baku untuk membuat halaman web.

CMS membuat halaman web dari database, sedangkan SSG tidak menggunakan database.

CMS akan membuat halaman web setiap kali ada permintaan, sedangkan SSG tidak.

CMS memiliki halaman admin untuk mengisi konten, sedangkan SSG tidak. SSG menggunakan teks editor.

Mana yang lebih cepat?

Kalau menurut saya SSG, karena server tidak berkerja begitu keras setiap ada request. Tinggal kirimkan saja halaman yang diminta.

Mana yang lebih murah?

Kalau menurut saya, SSG lebih murah. Karena kita tidak perlu sewa hosting untuk PHP dan database MySQL-nya.

Mana yang lebih mudah?

Dari segi kemudahan, tentunya CMS lebih mudah. Karena CMS memiliki halaman admin yang bersahabat. Sedangkan SSG lebih bersahabat bagi programmer.
